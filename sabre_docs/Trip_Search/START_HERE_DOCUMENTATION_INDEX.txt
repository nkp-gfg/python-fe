═══════════════════════════════════════════════════════════════════════════════
YOUR REQUIREMENT (SUMMARY)
═══════════════════════════════════════════════════════════════════════════════

BUSINESS NEED:
  "Get the list of passengers with their phone numbers for a specific 
   flight by passing the flight number, flight date and airline code"

INPUT (What you provide):
  • Flight Number        - e.g., "0001", "288"
  • Departure DateTime   - e.g., "2016-12-20T07:00:00"
  • Airline Code         - e.g., "ET", "AA"
  [Optional] Departure Airport - e.g., "JFK"

OUTPUT (What you receive):
  • PNR Locator          - Booking reference (6 characters)
  • Passenger Name       - First and last name
  • Seat Assignment      - Seat number (e.g., "12A")
  • Phone Numbers        - With type indicators (Home/Business/Mobile)

═══════════════════════════════════════════════════════════════════════════════
QUICK START (30-SECOND SUMMARY)
═══════════════════════════════════════════════════════════════════════════════

API ENDPOINT:   Trip_SearchRQ v4.5.0 (SOAP Web Service)
ENDPOINT URL:   http://usg.sabre.com/websvc
VIEW NAME:      TripSearchBasic (automatically includes phone numbers)
RESPONSE FORMAT: STL v1.19.0 (XML)

BASIC REQUEST:
  → Flight Number: Required
  → DepartureDateTime: Required (ISO 8601 format)
  → AirlineCode: Required
  → DepartureAirport: Recommended (3-letter IATA code)

RESPONSE CONTAINS:
  ✓ PNR Locators (booking references)
  ✓ Passenger names (FirstName, LastName)
  ✓ Seat assignments
  ✓ Phone numbers with type (Home/Business/Mobile)

KEY TAKEAWAYS - READ THIS FIRST
═══════════════════════════════════════════════════════════════════════════════

✓ API TO USE:         Trip_SearchRQ v4.5.0
✓ ENDPOINT:           http://usg.sabre.com/websvc
✓ PROTOCOL:           SOAP over HTTP/HTTPS
✓ BINDING:            TripSearchSoapBinding
✓ OPERATION:          TripSearchRQ
✓ RECOMMENDED VIEW:   TripSearchBasic
✓ RESPONSE FORMAT:    STL v1.19.0 (XML)

FOR YOUR SPECIFIC USE CASE:
  • Always use TripSearchBasic view (includes phones by default)
  • Required inputs: FlightNumber, DepartureDateTime, AirlineCode
  • Recommended: Add DepartureAirport for faster/more accurate results
  • Response includes: PNR Locator, Passenger names, Seats, Phone numbers
  • Maximum 800 PNRs per response (hard limit in v4.5.0)
  • Typical response time: < 5 seconds

CRITICAL POINTS:
  1. DateTime format MUST be ISO 8601: YYYY-MM-DDThh:mm:ss
  2. Authentication via SOAP security token required for all requests
  3. Phone numbers stored as numeric digits only
  4. Each PNR can have multiple passengers + multiple phone numbers
  5. Phone numbers are shared at PNR level (not per passenger)
  6. SearchType "ACTIVE" = current bookings, "INACTIVE" = past bookings
